# Product Intent

Updated: 2026-04-09
Purpose: Vibe Studio의 존재 이유, 제작 의도, 핵심 타깃 사용자를 고정하는 상위 문서

## 1. 제품 한 줄 정의
Vibe Studio는 막연한 생각이나 초안을 바로 결과물로 바꿔주는 생성기가 아니라, 사용자가 자신의 의도를 더 선명하게 이해하고 표현하도록 돕는 사고 구조화 학습 환경이다.

## 2. 문서의 역할과 우선순위
이 문서는 Vibe Studio의 상위 제품 의도 문서다.
PRD는 무엇을 만들지 설명하고, TRD는 어떻게 만들지 설명한다.
이 문서는 왜 만들고 누구를 위해 만드는지를 고정한다.

Rule:
- PRD와 TRD는 이 문서와 충돌하면 안 된다
- 새로운 기능 제안은 먼저 이 문서의 의도와 맞는지 점검해야 한다
- 제품 정의가 흔들릴 때는 이 문서를 먼저 기준으로 삼는다

## 3. 이 제품을 만드는 이유
많은 사람들은 AI를 쓰고 싶어 하지만, 무엇을 어떻게 말해야 원하는 결과에 가까워지는지 모르는 상태에서 멈춘다.
문제는 단순히 프롬프트를 잘 모른다가 아니라, 자기 의도와 요구를 구조화하는 언어를 아직 갖지 못했다는 점이다.

Vibe Studio는 이 지점을 돕기 위해 만든다.
사용자가 막연한 요청, 어설픈 초안, 정리되지 않은 아이디어를 가져오면, 시스템은 그것을 바로 완성품으로 밀어내기보다 먼저 구조화하고, 필요한 질문을 던지고, 더 나은 표현 방향을 비춰준다.

## 4. 제작 의도
이 제품의 목적은 사용자를 대신해 모든 것을 써주는 것이 아니다.
이 제품의 목적은 사용자가 AI와 더 잘 협업할 수 있도록, 자신의 생각을 더 명확하게 다루게 만드는 것이다.

나는 이 제품이 사용자를 수동적인 버튼 클릭 사용자로 만들기보다, 조금씩 더 잘 묻고, 더 잘 정리하고, 더 잘 검토할 수 있는 사람으로 돕기를 바란다.

## 5. 핵심 타깃 사용자
Vibe Studio의 핵심 타깃은 AI 입문자다.

이들은 보통 아래와 같은 상태에 있다.
- AI를 써보고 싶지만 무엇을 어떻게 요청해야 할지 잘 모른다
- 머릿속엔 생각이 있지만 그것을 구조적으로 설명하기 어렵다
- 프롬프트, 기획, 구조 설계, 검토의 차이를 아직 명확히 구분하지 못한다
- 대충 말해도 알아서 잘 만들어주길 바라면서도, 결과가 왜 아쉬운지 설명하긴 어렵다

이 제품은 이런 사용자가 자신의 생각을 더 분명하게 만들고, AI를 더 잘 쓰는 감각을 익히도록 돕는 것을 1차 목표로 한다.

## 6. 비핵심 사용자
이 제품의 1차 타깃이 아닌 사용자는 아래와 같다.
- 이미 고도화된 프롬프트 작성 습관을 가진 숙련 사용자
- 세밀한 파라미터 제어와 즉시 실행만 원하는 파워 유저
- 학습 과정이나 중간 확인 없이 빠른 결과만 원하는 사용자
- 전문적인 개발 또는 설계 도구를 기대하는 고급 기술 사용자

이 사용자들도 쓸 수는 있지만, 현재 제품 정의의 중심은 아니다.

## 7. 사용자가 경험해야 하는 변화
사용자는 Vibe Studio를 사용한 뒤 아래 변화 중 일부를 경험해야 한다.
- 막연한 요청을 더 구체적으로 말할 수 있게 된다
- 어떤 정보가 빠졌는지 스스로 더 잘 느끼게 된다
- 단순 생성보다 의도 확인과 구조화가 왜 중요한지 이해하게 된다
- 프롬프트, 기획, 구조 설계, 검토의 차이를 점점 감각적으로 익히게 된다
- AI에게 잘 말하는 법을 조금씩 배우게 된다

## 8. 제품이 절대 되면 안 되는 것
Vibe Studio는 아래와 같은 제품으로 흐르면 안 된다.
- 아무 생각 없이 결과만 뽑아주는 범용 생성기
- 점수와 경고만 많은 답답한 심사기
- 초보자에게 기술 용어를 강요하는 도구
- 사용자의 선택을 조용히 바꾸는 블랙박스 시스템
- 카드나 기능 이름을 먼저 이해해야만 쓸 수 있는 진입장벽 높은 제품

## 9. 제품 경험 원칙
- 사용자는 먼저 자유롭게 말할 수 있어야 한다
- 시스템은 먼저 이해한 바를 비춰주고, 필요할 때만 개입해야 한다
- 질문은 많기보다 정확해야 한다
- 마찰은 처벌처럼 느껴지지 않아야 한다
- 결과물 생성보다 의도 선명화가 앞서야 한다
- 사용자는 시스템에 끌려가는 느낌보다, 함께 정리해가는 느낌을 받아야 한다

## 10. 핵심 가치
Vibe Studio의 핵심 가치는 네 가지다.
- 구조화: 막연한 생각을 다룰 수 있는 형태로 바꾼다
- 학습: 사용자가 AI와 협업하는 감각을 익히게 한다
- 안내: 무엇이 부족한지, 왜 멈추는지 설명한다
- 명확화: 결과 이전에 의도를 더 선명하게 만든다

## 11. 성공의 징후
이 제품이 올바른 방향으로 작동하고 있다면 사용자는 이런 반응을 보일 가능성이 높다.
- 내가 뭘 원하는지 나도 더 잘 알게 됐다
- 왜 이 질문을 하는지 이해됐다
- 그냥 결과를 준 게 아니라, 생각 정리를 도와줬다
- 다음에는 내가 더 잘 요청할 수 있을 것 같다

## 12. 판단 기준
앞으로의 기능, UX, 엔진 설계는 아래 질문으로 점검한다.
- 이 결정이 초보 사용자의 막막함을 줄여주는가
- 이 결정이 사용자의 의도를 더 선명하게 만드는가
- 이 결정이 사용자를 더 수동적으로 만드는가, 더 능동적으로 만드는가
- 이 결정이 제품을 단순 생성기로 오해하게 만드는가
- 이 결정이 블랙박스성을 줄이고 이해 가능성을 높이는가
