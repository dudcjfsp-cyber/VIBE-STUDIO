# Prompt Help Learning Panel

Updated: 2026-04-19
Purpose: 프롬프트 도움 카드가 단순 복사용 출력이 아니라 학습용 해설을 함께 제공하도록 정보 구조와 노출 원칙을 고정한다.

## 1. Why This Exists
Vibe Studio의 프롬프트 도움 카드는 결과를 바로 쓰게 해주는 것만으로 끝나면 안 된다.
이 카드는 사용자가 왜 이런 입력이 더 안정적인 결과를 만드는지 이해하고, 다음에는 스스로 더 잘 요청하게 돕는 학습 장치여야 한다.

Rule:
- prompt help는 ready-to-use prompt만 출력하는 기능으로 축소하지 않는다
- prompt help는 결과 생성기보다 학습 보조 도구에 가깝게 다룬다
- 학습 해설은 최종 프롬프트를 압도하지 않아야 한다

## 2. Output Structure
프롬프트 도움 카드의 기본 구조는 2층으로 고정한다.

1. 바로 사용할 프롬프트
2. 학습 패널

Rule:
- 결과가 먼저, 해설이 나중이다
- 사용자는 먼저 복사해서 써볼 수 있어야 한다
- 학습 패널은 결과를 설명하고 재사용 감각을 남기는 역할을 맡는다

## 3. Learning Panel Sections
학습 패널은 아래 순서로 구성한다.

1. 이번에 같이 볼 포인트
2. 출력 형식 고정하기
3. 예시 없이 요청하기 (`Zero-shot`)
4. 예시를 보여주며 요청하기 (`Few-shot`)
5. 조건부 학습 포인트들

Rule:
- 항상 노출 섹션은 작은 수로 유지한다
- 조건부 섹션은 현재 입력 문제와 직접 연결될 때만 보여준다
- 내부 시스템 프롬프트 전체를 노출하지 않는다

## 4. Always-Visible Learning Points
항상 노출하는 대표 기법은 아래 세 가지로 고정한다.

1. 출력 형식 고정하기
2. 예시 없이 요청하기 (`Zero-shot`)
3. 예시를 보여주며 요청하기 (`Few-shot`)

Reason:
- 초보 사용자가 가장 빨리 이해할 수 있는 축이다
- 결과 일관성과 예시 사용 여부를 함께 배울 수 있다
- 과도한 기법 나열 없이도 학습 경험을 만들 수 있다

## 5. Conditional Learning Points
아래 기법은 입력 문제가 실제로 드러날 때만 보여준다.

1. 역할을 정하고 요청하기
2. 범위를 좁혀서 요청하기
3. 겹치는 지시 정리하기
4. 단계로 나눠 요청하기

Rule:
- 역할 충돌이 없으면 역할 기법을 억지로 보여주지 않는다
- 범위가 이미 선명하면 범위 제한 기법을 보여주지 않는다
- 모든 기법을 매번 보여주지 않는다

## 6. Naming Rule
기법 이름은 한글 설명형 이름을 먼저 보여준다.
업계 용어 연결이 필요한 경우에만 괄호 안에 영문 프롬프트 엔지니어링 용어를 병기한다.

Examples:
- 예시 없이 요청하기 (`Zero-shot`)
- 예시를 보여주며 요청하기 (`Few-shot`)

Rule:
- 영어 용어를 제목 앞으로 세우지 않는다
- 사용자가 의미를 먼저 이해하고, 용어는 나중에 연결되게 한다
- 한글 이름은 사용자가 실제로 취하는 행동처럼 들려야 한다

## 7. Card Copy Format
각 학습 포인트는 아래 네 줄 포맷을 기본으로 한다.

- 기법 이름
- 언제 쓰나
- 이번 적용
- 왜 중요했나

Rule:
- 설명은 짧고 plain language로 쓴다
- 기술 용어 설명보다 상황 설명을 우선한다
- 정답 공식처럼 말하지 않는다

## 8. What Not To Show
아래 요소는 학습 패널에 직접 노출하지 않는다.

- 내부 시스템 프롬프트 전체
- 불필요하게 긴 프롬프트 엔지니어링 백과사전
- 현재 입력과 관련 없는 기법 목록
- 사용자가 그대로 외워야 할 정답 공식처럼 보이는 표현

## 9. Judgement Rule
프롬프트 도움 카드 변경안은 아래 질문으로 점검한다.

- 이 해설이 사용자를 더 능동적으로 만드는가
- 사용자가 왜 이렇게 바뀌었는지 이해할 수 있는가
- 다음에 직접 써볼 감각을 남기는가
- 결과보다 해설이 더 무겁게 느껴지지 않는가
- 영어 용어를 몰라도 핵심을 이해할 수 있는가

## 10. Scope Note
이 문서는 정보 구조, 노출 원칙, 학습 목적을 다룬다.
세부 UI 배치, 실제 문장 카피, 세션별 예시 문구는 이 문서의 범위를 넘는다.

Rule:
- 제품 원칙은 이 문서에 남긴다
- 화면 카피 미세 조정은 별도 구현 문서나 작업 스레드에서 다룬다
